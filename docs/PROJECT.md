# PROJECT.md

> Telegram CRM для дитячої ігрової кімнати.

---

# 1. Мета проєкту

Створити систему керування дитячою ігровою кімнатою через Telegram.

Система повинна:

- вести облік клієнтів;
- вести картки дітей;
- вести історію відвідувань;
- автоматично рахувати оплату;
- допомагати адміністраторам;
- збирати статистику;
- працювати швидко та просто.

---

# 2. Основні принципи

- Один Telegram акаунт = один користувач.
- Один користувач може мати кілька дітей.
- Кожна дитина має власну віртуальну картку.
- Кожне відвідування зберігається в історії.
- Будь-яка дія користувача логується.
- Всі розрахунки виконуються автоматично.
- Один користувач може бути лише одного типу (`PARENT` або `EMPLOYEE`).
- Права працівників визначаються полем `employeeRole`.
- `SUPER_ADMIN` успадковує всі можливості `ADMIN`.

---

# 3. Типи користувачів

## PARENT

Клієнт ігрової кімнати.

Може:

- переглядати своїх дітей;
- додавати дітей;
- приєднуватися до картки дитини;
- починати візит;
- переглядати історію відвідувань;
- переглядати ціни;
- переглядати інформацію про кімнату.

---

## EMPLOYEE

Працівник ігрової кімнати.

Рівень доступу визначається полем `employeeRole`.

### ADMIN

Може:

- знаходити картки дітей;
- створювати картки вручну;
- редагувати картки;
- підтверджувати початок візиту;
- завершувати візити;
- переглядати активні візити;
- переглядати статистику за день.

### SUPER_ADMIN

Успадковує всі можливості `ADMIN`.

Додатково може:

- переглядати всіх клієнтів;
- переглядати всі картки дітей;
- переглядати доходи;
- переглядати розширену статистику;
- керувати адміністраторами;
- змінювати налаштування системи.

## Програма лояльності

Після **9 платних відвідувань** клієнт отримує:

- ✔ 1 безкоштовну годину.

Після використання безкоштовного відвідування цикл починається знову.

Безкоштовне відвідування також зберігається в історії, але не збільшує лічильник платних відвідувань.

# 6. Структура бази даних

## User

Зберігає інформацію про всіх користувачів.

Основні поля:

- id
- telegramId
- phone
- firstName
- lastName
- userType
- employeeRole
- createdAt

---

## Child

Картка дитини.

Основні поля:

- id
- cardNumber
- firstName
- lastName
- birthday
- loyaltyVisits
- freeVisitBalance
- totalVisits
- freeVisitsUsed
- createdAt

---

## ChildAccess

Зв'язок між дитиною та її батьками.

Дозволяє:

- одному з батьків мати багато дітей;
- одній дитині мати кількох батьків.

---

## Visit

Зберігає інформацію про кожне відвідування.

Основні поля:

- id
- childId
- status
- requestedByUserId
- startedByAdminId
- finishedByAdminId
- startTime
- endTime
- durationMinutes
- totalPrice
- isFreeVisit

---

## Settings

Загальні налаштування системи.

Містить:

- тарифи;
- програму лояльності;
- інформацію про кімнату;
- графік роботи.

---

## AuditLog

Журнал усіх дій у системі.
