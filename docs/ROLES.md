# ROLES.md

> Система ролей та контролю доступу (RBAC)

---

# Архітектура

```text
                    USER
                      │
                  userType
                      │
        ┌─────────────┴─────────────┐
        │                           │
     PARENT                   EMPLOYEE
                                        │
                                        ▼
                                 employeeRole
                                        │
                           ┌────────────┴────────────┐
                           ▼                         ▼
                        ADMIN                 SUPER_ADMIN
```

---

# Основні принципи

- PARENT — окремий тип користувача.
- EMPLOYEE — тип користувача для працівників.
- Права працівників визначаються полем `employeeRole`.
- SUPER_ADMIN успадковує всі можливості ADMIN.
- SUPER_ADMIN не успадковує меню PARENT.
- Усі перевірки доступу виконуються через `userType` та `employeeRole`.

# UserType

```text
PARENT

EMPLOYEE
```

---

## PARENT

Відвідувач ігрової кімнати.

### Може

- переглядати своїх дітей;
- додавати дітей;
- редагувати інформацію про своїх дітей;
- переглядати історію відвідувань;
- переглядати активний візит;
- створювати запит на початок візиту;
- запросити другого з батьків;
- переглядати ціни;
- переглядати інформацію про ігрову кімнату.

### Не може

- бачити інших клієнтів;
- бачити інших дітей;
- підтверджувати початок візиту;
- завершувати візити;
- переглядати доходи;
- керувати адміністраторами;
- змінювати налаштування системи.

---

## EMPLOYEE

Працівник ігрової кімнати.

Рівень доступу визначається полем `employeeRole`.

---

# EmployeeRole

```text
ADMIN

SUPER_ADMIN
```

---

## ADMIN

Адміністратор зміни.

### Може

- знаходити картки дітей;
- переглядати всі картки дітей;
- створювати картки вручну;
- редагувати картки;
- підтверджувати початок візиту;
- завершувати візити;
- переглядати активні візити;
- переглядати статистику за день;
- створювати нового клієнта.

### Не може

- переглядати доходи;
- переглядати аналітику;
- керувати адміністраторами;
- змінювати налаштування системи.

---

## SUPER_ADMIN

Власник бізнесу.

SUPER_ADMIN автоматично має всі можливості ADMIN.

### Додатково може

- переглядати Dashboard;
- переглядати всіх клієнтів;
- переглядати всі картки;
- переглядати доходи;
- переглядати аналітику;
- керувати адміністраторами;
- змінювати тарифи;
- змінювати налаштування системи;
- переглядати журнал подій.

# Перевірка доступу

## PARENT

```ts
user.userType === UserType.PARENT
```

---

## Будь-який працівник

```ts
user.userType === UserType.EMPLOYEE
```

---

## ADMIN

```ts
user.userType === UserType.EMPLOYEE &&
user.employeeRole === EmployeeRole.ADMIN
```

---

## SUPER_ADMIN

```ts
user.userType === UserType.EMPLOYEE &&
user.employeeRole === EmployeeRole.SUPER_ADMIN
```

---

# Принципи

1. PARENT і EMPLOYEE — різні типи користувачів.
2. EMPLOYEE обов'язково має `employeeRole`.
3. SUPER_ADMIN успадковує всі права ADMIN.
4. SUPER_ADMIN використовує власну панель керування, а не клієнтський інтерфейс.
5. Система легко розширюється додаванням нових ролей (`MANAGER`, `CASHIER` тощо) без зміни архітектури.
